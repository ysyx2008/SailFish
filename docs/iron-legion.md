# 钢铁军团 (Iron Legion)

> AI 多终端协作执行系统

## 概述

钢铁军团是旗鱼终端的核心功能之一，采用 **Master-Worker 架构**，让 AI 能够同时协调多台服务器执行运维任务。用户只需用自然语言描述任务，AI 会自动识别目标服务器、连接终端、派发任务、收集结果并生成分析报告。

## 核心能力

| 能力 | 说明 |
|------|------|
| 🖥️ **多终端连接** | 同时连接多个 SSH 服务器或本地终端 |
| 🤖 **智能调度** | AI 自动识别目标服务器，分配任务 |
| ⚡ **并行执行** | 支持多台服务器同时执行任务，提高效率 |
| 📊 **结果汇总** | 自动收集各服务器执行结果，生成分析报告 |
| 🔒 **安全确认** | 危险命令需要用户确认，支持多种确认策略 |

## 架构说明

```
┌─────────────────────────────────────────────────────────────┐
│                     Master Agent                             │
│  负责任务分析、服务器选择、任务派发、结果汇总                    │
└─────────────────────────────────────────────────────────────┘
                              │
           ┌──────────────────┼──────────────────┐
           │                  │                  │
           ▼                  ▼                  ▼
    ┌─────────────┐   ┌─────────────┐   ┌─────────────┐
    │ Worker Agent│   │ Worker Agent│   │ Worker Agent│
    │  (服务器 A)  │   │  (服务器 B)  │   │  (本地终端)  │
    └─────────────┘   └─────────────┘   └─────────────┘
```

- **Master Agent**：协调者，负责理解用户任务、选择目标服务器、派发子任务、汇总结果
- **Worker Agent**：执行者，在各个终端中执行具体命令，返回执行结果

## 可用工具

钢铁军团的 Master Agent 拥有以下 8 个专用工具：

### 1. `list_available_hosts` - 列出可用主机

列出会话管理器中所有可用的 SSH 主机配置。

**返回信息**：
- `hostId`: 主机配置的唯一标识
- `name`: 主机显示名称
- `host`: 主机地址
- `port`: SSH 端口
- `username`: 用户名
- `group`: 所属分组名称（可用于匹配"生产服务器"、"测试环境"等）
- `tags`: 标签列表

**使用场景**：
- 任务开始时先调用此工具了解可用主机
- 根据用户任务描述匹配相关主机（如"生产服务器"→匹配分组名包含"生产"或"prod"的主机）

---

### 2. `connect_terminal` - 连接终端

创建新的终端连接（本地或 SSH）。

**参数**：
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `type` | `'local'` \| `'ssh'` | ✅ | 终端类型 |
| `host_id` | string | SSH 时必填 | 主机配置 ID，从 list_available_hosts 获取 |
| `alias` | string | ✅ | 终端别名，用于后续引用（如 "prod-web-1"） |

**返回**：
- `terminalId`: 新创建的终端 ID
- `alias`: 终端别名
- `type`: 终端类型
- `connected`: 是否连接成功

**使用场景**：
- 需要在本地执行脚本或汇总分析 → `type='local'`
- 需要连接远程服务器 → `type='ssh'`
- 混合任务可以同时创建两种类型的终端

---

### 3. `dispatch_task` - 派发任务

向指定终端派发任务，由该终端的 Worker Agent 执行。

**参数**：
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `terminal_id` | string | ✅ | 终端 ID 或别名 |
| `task` | string | ✅ | 任务描述（自然语言，Worker Agent 会理解并执行） |
| `wait_for_result` | boolean | ❌ | 是否等待执行完成，默认 true |

**返回**：
- `success`: 任务是否成功完成
- `result`: 执行结果（命令输出或分析结果）
- `error`: 错误信息（如果失败）
- `duration`: 执行耗时（毫秒）

**注意**：
- 任务描述应清晰具体，Worker Agent 会自行决定执行哪些命令
- 危险命令会触发确认机制（取决于用户选择的确认策略）

---

### 4. `parallel_dispatch` - 并行派发

并行向多个终端派发相同任务，这是批量执行的高效方式。

**参数**：
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `terminal_ids` | string[] | ✅ | 终端 ID 或别名的数组 |
| `task` | string | ✅ | 任务描述（所有终端执行相同任务） |

**返回**：
- `results`: 各终端的执行结果数组
  - `terminalId`: 终端 ID
  - `terminalName`: 终端名称
  - `success`: 是否成功
  - `result`: 执行结果
  - `error`: 错误信息
  - `duration`: 执行耗时

**注意**：
- 适用于需要在多台服务器执行相同检查的场景
- 实际并行数受 `maxParallelWorkers` 配置限制
- 如果有危险命令，会触发批量确认（可一次确认应用到所有终端）

---

### 5. `get_task_status` - 查询任务状态

查询某个终端任务的执行状态。

**参数**：
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `terminal_id` | string | ✅ | 终端 ID 或别名 |

**返回**：
- `status`: 状态（`connecting` / `idle` / `running` / `completed` / `failed` / `timeout`）
- `currentTask`: 当前执行的任务（如果正在执行）
- `result`: 执行结果（如果已完成）
- `error`: 错误信息（如果失败）
- `duration`: 已执行时间（毫秒）

**使用场景**：
- `dispatch_task` 设置 `wait_for_result=false` 后轮询查询
- 监控长时间运行的任务进度

---

### 6. `collect_results` - 收集结果

收集并汇总多个终端的执行结果，适用于任务执行完成后的结果分析。

**参数**：
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `terminal_ids` | string[] | ❌ | 要收集结果的终端 ID 数组，不指定则收集所有 |
| `format` | `'table'` \| `'list'` \| `'summary'` | ❌ | 输出格式 |

**输出格式说明**：
- `table`: 表格形式，适合对比
- `list`: 列表形式，详细输出
- `summary`: 仅统计汇总

**返回**：
- `totalCount`: 总任务数
- `successCount`: 成功数
- `failedCount`: 失败数
- `results`: 各终端的详细结果
- `formatted`: 按指定格式的输出文本

---

### 7. `close_terminal` - 关闭终端

关闭指定的终端连接。

**参数**：
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `terminal_id` | string | ✅ | 终端 ID 或别名 |

**返回**：
- `success`: 是否成功关闭

**使用场景**：
- 任务完成后清理不再需要的终端连接
- 释放系统资源

**注意**：
- 关闭后终端 ID 将失效
- 如果终端有任务正在执行，会先停止任务

---

### 8. `analyze_and_report` - 分析报告

分析收集的结果并生成报告。调用此工具表示任务执行完成。

**参数**：
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `findings` | string[] | ✅ | 主要发现列表 |
| `recommendations` | string[] | ✅ | 建议措施列表 |
| `severity` | `'info'` \| `'warning'` \| `'critical'` | ✅ | 整体严重程度 |

**严重程度说明**：
- `info`: 正常，无需处理
- `warning`: 需关注，建议处理
- `critical`: 需立即处理

**使用场景**：
- 所有终端任务执行完成后
- 汇总分析结果并给出建议
- 结束本次任务

---

## 工作流程

```
用户输入任务
    │
    ▼
┌─────────────────────────────────────┐
│ 1. list_available_hosts             │  ← 了解可用主机
└─────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────┐
│ 2. connect_terminal (多次)          │  ← 连接目标服务器
└─────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────┐
│ 3. parallel_dispatch / dispatch_task│  ← 派发任务执行
└─────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────┐
│ 4. collect_results                  │  ← 收集执行结果
└─────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────┐
│ 5. analyze_and_report               │  ← 生成分析报告
└─────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────┐
│ 6. close_terminal (可选)            │  ← 清理终端连接
└─────────────────────────────────────┘
```

## 使用场景示例

### 示例 1：检查磁盘使用

**用户输入**：
> 检查所有生产服务器的磁盘使用情况，标记超过 80% 的

**AI 执行流程**：
1. 调用 `list_available_hosts` 获取主机列表
2. 筛选分组名包含"生产"的主机
3. 调用 `connect_terminal` 连接这些服务器
4. 调用 `parallel_dispatch` 执行 "检查磁盘使用率"
5. 调用 `collect_results` 汇总结果
6. 调用 `analyze_and_report` 生成报告，标注超过 80% 的磁盘

### 示例 2：服务健康检查

**用户输入**：
> 检查所有 Web 服务器上的 nginx 是否正常运行

**AI 执行流程**：
1. 筛选 Web 服务器
2. 并行检查 nginx 服务状态
3. 汇总哪些服务器 nginx 正常，哪些异常
4. 给出建议（如：某服务器 nginx 未运行，建议重启）

### 示例 3：批量执行命令

**用户输入**：
> 在所有测试服务器上清理 /tmp 下超过 7 天的文件

**AI 执行流程**：
1. 筛选测试环境服务器
2. 派发清理任务（因为是删除操作，会触发确认）
3. 用户确认后执行
4. 汇总清理结果

## 确认策略

钢铁军团支持三种确认策略，用户可在界面上选择：

| 策略 | 说明 | 适用场景 |
|------|------|----------|
| **审慎模式** | 每个危险命令都需要确认 | 敏感环境，需要完全掌控 |
| **批量确认** | 相同命令在多台服务器执行时，一次确认应用到所有 | 日常使用，平衡效率和安全 |
| **自由模式** | 自动执行，不做任何确认 ⚠️ | 完全信任 AI，了解风险 |

## 安全机制

1. **命令风险评估**：Worker Agent 会对每个命令进行风险评估
   - `safe`: 只读命令，自动执行
   - `moderate`: 中等风险，根据策略确认
   - `dangerous`: 高风险（如 rm、shutdown），必须确认
   - `blocked`: 禁止执行的危险命令

2. **执行可见**：所有命令都在终端中执行，用户可以看到完整的输入输出

3. **随时中止**：用户可以随时点击"停止"按钮中止任务执行

4. **超时保护**：长时间无响应的任务会自动超时

## 配置选项

| 选项 | 说明 | 默认值 |
|------|------|--------|
| `maxParallelWorkers` | 最大并行执行数 | 5 |
| `workerTimeout` | Worker 超时时间（毫秒） | 300000 (5分钟) |
| `autoCloseTerminals` | 任务完成后自动关闭终端 | false |
| `confirmStrategy` | 确认策略 | batch |
