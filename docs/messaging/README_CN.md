# IM 集成指南

中文 | [English](./README.md)

本文档介绍如何将旗鱼终端的 AI Agent 接入钉钉、飞书、企业微信、Slack 和 Telegram，实现在 IM 中直接与 AI 对话。

连接成功后，你可以在 IM 中像跟同事聊天一样与 AI Agent 交互——发一条消息，Agent 就会在你的电脑上执行任务并回复结果。

---

## 工作原理

```
IM 用户发送消息
       │
       ▼
  钉钉/飞书/企业微信/Slack/Telegram 服务器
       │ (WebSocket 长连接 / Long Polling 推送)
       ▼
  旗鱼终端 IM 服务
       │
       ▼
  本地 AI Agent 处理
       │
       ▼
  通过 API 回复到 IM
```

- **多数平台无需公网服务器**：钉钉、飞书、Slack、Telegram 使用 WebSocket 长连接或长轮询，由客户端主动连接，无需暴露端口或配置域名。**企业微信**使用 HTTP 回调，需提供公网可访问的回调 URL（可用内网穿透等）。
- **消息在本地处理**：所有 AI 推理和工具调用都在你的电脑上执行。
- **独立于 Gateway**：IM 集成不依赖远程访问（Gateway）服务，可以单独使用。

## 在旗鱼终端中连接

1. 打开旗鱼终端，进入 **设置** → **远程访问**
2. 在 **IM 集成** 区域，展开对应平台的卡片（钉钉 / 飞书 / 企业微信 / Slack / Telegram）
3. 填入上方配置指南中获取的凭证：
   - **钉钉**：ClientID + ClientSecret
   - **飞书**：App ID + App Secret
   - **企业微信**：企业 ID、Corp Secret、AgentId、回调 Token、EncodingAESKey、回调端口
   - **Slack**：Bot Token（xoxb-...）+ App-Level Token（xapp-...）
   - **Telegram**：Bot Token
4. 点击 **连接**，等待状态变为 ✅ **已连接**
5. 可选：勾选 **启动时自动连接**，下次打开旗鱼终端会自动连上

## 使用方式

### 私聊

直接在 IM 中搜索你创建的机器人名称，发起私聊，发送文字消息即可。

### 群聊

1. 将机器人添加到群聊中
2. **@机器人** 后输入你的消息
3. 机器人会在群内回复

### 支持的消息类型

- **输入**：目前仅支持文本消息
- **输出**：支持纯文本、Markdown 格式回复和文件发送
  - 钉钉、企业微信以 Markdown 消息发送，飞书以交互卡片形式发送，Slack 以 mrkdwn 格式发送，Telegram 以 Markdown 格式发送
  - AI 可以将机器上的文件直接发送到聊天中（钉钉/企业微信限 20MB，飞书限 30MB，Slack 限 1GB，Telegram 限 50MB）

### 文件发送

AI Agent 可以将本地文件通过 IM 机器人发送给你。典型场景：

- 你让 AI 查看某个日志文件，AI 可以直接把文件发过来
- AI 生成了报告、截图、导出文件后，自动发送到聊天
- 需要拿到服务器上的配置文件时，直接告诉 AI "把 xxx 文件发给我"

## 常见问题

### 连接失败

- 检查凭证是否正确（注意不要有多余空格）
- 确认应用已发布上线（钉钉/飞书需要发布版本，Slack 需要 Install to Workspace，Telegram 创建机器人后即可使用）
- 检查网络是否能访问对应平台的服务器

### 机器人不回复消息

- 确认旗鱼终端正在运行且 IM 连接状态为"已连接"
- 群聊中需要 **@机器人** 才会触发
- 检查旗鱼终端的 AI 模型配置是否正常

### 消息被截断

IM 平台对单条消息长度有限制。Agent 回复过长时会自动截断并提示"内容已截断"。

### 按平台查看

- **Slack**："向此应用发送消息的功能已关闭" → 见 [Slack 配置第 5 步](./slack_CN.md#第-5-步启用-app-home-消息功能)
- **飞书**："应用未建立长连接" → 见 [飞书配置第 5 步](./feishu_CN.md#第-5-步先连接旗鱼终端)
- **企业微信**：收不到回调消息 → 见 [企业微信配置](./wecom_CN.md#常见问题收不到回调消息)（需确保回调 URL 可从公网访问）

### 凭证安全

所有凭证仅保存在本地设备上，不会上传到任何服务器。请确保你的设备安全，避免凭证泄露。
