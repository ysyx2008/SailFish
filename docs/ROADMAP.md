## 一、记忆系统增强

### 背景
当前记忆系统存在以下问题：
- 只能清空全部记忆，无法选择性删除
- 记忆可能过时、错误或不再适用
- 用户纠正信息时，AI 无法自动更新旧记忆

### 计划功能

#### 阶段 1：基础管理功能
- [ ] **批量删除记忆**：在记忆标签页添加复选框多选，支持批量删除
- [ ] **编辑记忆**：允许用户直接修改已有记忆内容
- [ ] **按主机批量删除**：一键清空某个主机的所有记忆

#### 阶段 2：智能辅助
- [ ] **使用追踪**：记录 `lastUsedAt`（最后召回时间）和 `usageCount`（使用次数）
- [ ] **新鲜度指示**：基于创建时间和使用情况显示「新/正常/陈旧」标签
- [ ] **智能排序**：召回时按 `相关性 × 新鲜度 × 使用频率` 综合排序
- [ ] **陈旧提醒**：超过 30 天未使用的记忆显示"可能过时"

#### 阶段 3：AI 辅助管理
- [ ] **update_memory 工具**：AI 识别用户纠正信息时，自动更新相关旧记忆
- [ ] **记忆冲突检测**：检测语义相似但内容矛盾的记忆
- [ ] **AI 辅助清理**：定期扫描，识别可能过时的记忆供用户确认

## 三、定时任务调度系统（优先级：高）

### 背景
用户希望能够设定任务在指定时间自动执行，无需手动启动 Agent。例如：
- 每天凌晨 3 点执行备份检查
- 每周一上午 9 点生成周报
- 某个时间点一次性执行部署任务

### 核心功能

#### 阶段 1：基础调度能力
- [ ] **SchedulerService 服务**：基于 node-schedule 实现任务调度
- [ ] **任务持久化**：存储到本地 JSON，应用重启后自动恢复
- [ ] **单次执行**：设定未来某个时间点执行任务
- [ ] **周期执行**：支持 cron 表达式定义周期

#### 阶段 2：Agent 工具集成
- [ ] **schedule_task 工具**：AI 可创建定时任务
- [ ] **list_scheduled_tasks 工具**：列出所有定时任务
- [ ] **cancel_scheduled_task 工具**：取消/删除任务

#### 阶段 3：前端管理界面
- [ ] **独立页面 ScheduledTasksPage**：任务列表、创建、编辑、删除
- [ ] **欢迎页入口**：与"本地终端"、"SSH连接"并列的快速入口
- [ ] **任务执行历史**：查看每次执行的结果和日志
- [ ] **主机选择**：支持指定任务在哪台主机上执行

### 技术要点
- 使用 `node-schedule` 进行 cron 调度
- 通过 IPC 连接前端与主进程调度服务
- 任务触发时自动创建终端/SSH 连接并启动 Agent
