# 数据安全与隐私

旗鱼作为 AI Agent 应用，会将你的对话内容、终端输出等信息发送给 AI 服务商进行处理。本文帮助你了解各服务商的数据政策，以便做出合适的选择。

## 为什么需要关注

使用旗鱼时，发送给 AI 的数据可能包含：

- **终端输出** — IP 地址、文件路径、进程信息
- **命令历史** — 可能包含密码、密钥、Token
- **服务器信息** — SSH 连接参数、主机名、配置
- **业务内容** — 代码、日志、文档

这些信息的敏感程度远高于普通聊天对话，因此选择服务商时需要了解其数据处理方式。

## 各服务商政策一览

> 以下信息基于各服务商截至 2026 年 3 月的公开政策，政策可能随时更新。

### 国际服务商

| 服务商 | API 数据用于训练 | 数据保留期 | 零保留选项 |
|---|---|---|---|
| **Anthropic**（Claude） | 默认**不用** | 30 天自动删除 | 支持 |
| **OpenAI**（GPT-4o / o3） | 默认**不用**（2023.3 起） | 30 天 | 支持（需申请） |
| **Google**（Gemini） | 付费 API **不用**；免费层可能用 | 55 天 | 不支持 |

**共同特点**：
- API 层面默认不将用户数据用于模型训练，有明确的合同承诺
- 用户拥有输入和输出的所有权
- 提供传输加密（TLS）和静态加密（AES-256）

### 国内服务商

| 服务商 | API 数据用于训练 | Opt-out 方式 | 政策明确度 |
|---|---|---|---|
| **DeepSeek** | 默认可能用（去标识化） | 关闭"数据优化体验"开关 | 较明确 |
| **豆包**（字节） | 默认可能用（去标识化） | 关闭"帮助模型改进"开关 | 较明确 |
| **文心一言**（百度） | 去标识化后可用 | 仅记忆簿需许可，普通对话不明确 | 一般 |
| **通义千问**（阿里） | 未直接说明 | 未明确 | 较模糊 |
| **智谱 GLM** | 未明确说明 | 未明确 | 较模糊 |
| **Kimi**（月之暗面） | 可能用于"改进服务" | 未明确 | 较模糊 |

**共同特点**：
- 多数采用"默认可能用于训练，用户可 opt-out"的模式
- 均承诺去标识化 / 匿名化处理
- 部分服务商的 API 和消费端政策边界不够清晰

## 安全等级参考

从数据保护角度，API 调用场景的安全等级大致为：

```
最安全 ────────────────────────────────────── 需谨慎
Anthropic ≈ OpenAI > Google(付费) > DeepSeek ≈ 豆包 > 智谱 > 百度 ≈ 通义 ≈ Kimi
```

## 法律环境

### 中国

中国是全球最早对生成式 AI 专门立法的国家之一：

- **《生成式人工智能服务管理暂行办法》**（2023 年 8 月施行）：使用个人信息训练模型**必须取得个人同意**；不得非法留存可识别用户身份的输入信息
- **《个人信息保护法》**：同意必须"充分知情、自愿、明确"；用户有权撤回同意；禁止捆绑同意

但据 2024 年南都测评，国内多数平台在实际执行中仍存在告知不充分、退出机制不便捷等问题。

### 欧盟

- **GDPR**：数据最小化、目的限制——为一个目的收集的数据不能未经同意用于训练
- **EU AI Act**：2026 年 8 月全面生效，要求公开训练数据摘要、粒度化的用户同意

### 美国

- 联邦层面无综合性 AI 法案
- 加州有《AI 训练数据透明度法案》，要求公开训练数据来源
- 超过 30 个州已制定或正在推进 AI 相关立法

## 使用建议

### 按安全需求选择

| 场景 | 推荐方案 |
|---|---|
| 金融、政务等高敏感 | 本地部署开源模型（DeepSeek、Qwen 等） |
| 企业内部、生产服务器 | Anthropic / OpenAI 付费 API |
| 个人开发、学习 | 国际或国内 API 均可（确保关闭训练开关） |
| 公开信息处理 | 任意服务商 |

### 通用安全习惯

1. **优先使用 API 而非消费端**：API 的数据保护政策普遍优于 ChatGPT、文心一言 App 等消费端产品
2. **使用国内服务时检查设置**：确认关闭"数据用于模型改进"类开关
3. **避免明文密钥**：不要在对话中直接粘贴密码、API Key 等敏感信息
4. **定期审查政策**：服务商隐私政策可能更新，建议每半年复查
