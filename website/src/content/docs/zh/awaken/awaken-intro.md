# 觉醒模式入门

觉醒模式让 AI 不再只是被动等你发消息——你可以设定「关切」，让 Agent 在特定条件下自动执行任务并推送结果。比如每天早上自动汇报服务器状态，收到重要邮件时立即通知，或监控日志文件发现异常时告警。

## 核心概念

| 概念 | 说明 |
|------|------|
| **关切（Watch）** | 一条"关注配置"——定义什么时候做、做什么、结果送哪里 |
| **触发器（Trigger）** | 关切的触发条件，支持定时、间隔、文件变化、邮件到达等 |
| **传感器（Sensor）** | 底层事件感知器，不消耗 AI，只负责检测事件的发生 |
| **输出渠道** | 结果投递方式：应用内对话、IM 推送、系统通知或静默记录 |

## 支持的触发器

| 触发器类型 | 说明 | 示例 |
|-----------|------|------|
| 定时 (cron) | Cron 表达式 | 每天 9:00、每周一 10:00 |
| 间隔 (interval) | 固定间隔执行 | 每 30 分钟、每 2 小时 |
| 文件变化 | 监控指定文件或目录的增删改 | /var/log/error.log 发生变化 |
| 新邮件 | 收到新邮件时触发 | 来自 boss@company.com 的邮件 |
| 日历事件 | 日程开始前 N 分钟触发 | 会议开始前 15 分钟提醒 |
| Webhook | 外部系统通过 HTTP 回调触发 | CI/CD 完成时通知 |
| 手动 | 仅在觉醒面板中手动触发 | 按需执行的任务 |

## 创建关切

有两种方式创建关切：

### 方式一：和 Agent 对话（推荐）

直接用自然语言描述你的需求，Agent 会自动创建关切：

```
每天早上 9 点检查服务器状态并通过钉钉通知我
```

```
监控 /var/log/nginx/error.log，发现新的 500 错误时通过飞书提醒我
```

```
每周五下午 4 点生成本周工作总结
```

### 方式二：觉醒面板

1. 点击左侧边栏的觉醒面板图标
2. 点击「新建关切」
3. 选择一个内置模板或从头配置
4. 设置触发器、任务描述和输出渠道

## 内置模板

旗鱼提供了 6 个常用场景的模板，选择后一键创建：

| 模板 | 功能 |
|------|------|
| 每日晨间简报 | 每天早上推送天气、日程、重要邮件和新闻摘要 |
| 日历事件提醒 | 日程开始前自动提醒，附带上下文信息 |
| 邮件智能分类 | 新邮件自动分类，重要邮件立即通知 |
| 日志文件监控 | 监控日志变化，发现错误或异常时告警 |
| 服务器定期巡检 | 定期检查 CPU、内存、磁盘和服务状态 |
| 文件变更通知 | 监控目录文件变更并通知 |

## 觉醒面板

点击左侧边栏的觉醒图标打开面板，可以查看：

- **所有关切**的状态（运行中/暂停/错误）
- **执行历史**：每次触发的时间和结果
- **传感器事件**：底层检测到的事件记录

你可以在面板中暂停、恢复、编辑或删除关切。

## 使用建议

- 从模板开始，根据需要调整触发时间和任务描述
- 关切使用独立的 Agent，不会影响你的日常对话
- 设置输出渠道为 IM 可以在手机端接收通知
- 关切的 AI 执行前会先判断「该不该做」，避免不必要的执行
