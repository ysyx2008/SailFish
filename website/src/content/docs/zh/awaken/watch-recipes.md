# 常用关切配置

这里整理了几个常见场景的关切配置案例，你可以直接参考或通过和 Agent 对话来创建。

## 服务器健康巡检

**场景**：每天定时检查服务器状态，发现问题及时通知。

```
请创建一个关切：
- 每天早上 8:30 和下午 5:00 各执行一次
- 检查服务器的 CPU 使用率、内存占用、磁盘空间、关键服务运行状态
- 如果有异常（CPU > 80%、磁盘 > 90%、服务停止），通过钉钉通知我
- 如果一切正常，只在应用内记录，不发通知打扰我
```

## 日志异常监控

**场景**：实时监控应用日志，发现错误立即告警。

```
请创建一个关切：
- 监控 /var/log/app/error.log 文件的变化
- 有新内容时分析是否包含严重错误（如 OOM、数据库连接失败、超时）
- 如果发现严重错误，立即通过飞书通知我并附上错误详情和初步分析
- 普通 warning 级别的不用管
```

## 每日晨间简报

**场景**：每天早上自动推送今天需要关注的信息。

```
请创建一个关切：
- 每天早上 8:00 执行
- 内容包括：今天的日程安排、未读的重要邮件摘要、服务器状态概览
- 通过企业微信推送给我
```

## 定时备份检查

**场景**：检查自动备份是否正常完成。

```
请创建一个关切：
- 每天凌晨 3:00 执行（备份脚本在 2:00 运行）
- 检查 /backup/ 目录下是否有今天日期的备份文件
- 检查备份文件大小是否正常（和昨天的差距不超过 20%）
- 如果备份失败或异常，立即通过钉钉通知我
```

## CI/CD 部署通知

**场景**：通过 Webhook 接收 CI/CD 管道的完成通知。

```
请创建一个关切：
- 触发方式为 Webhook
- 收到 Webhook 时解析部署状态
- 如果部署成功，检查应用是否正常启动，然后通过飞书通知团队
- 如果部署失败，分析错误原因并给出修复建议
```

## 文件变更追踪

**场景**：监控重要配置文件的变更。

```
请创建一个关切：
- 监控 /etc/nginx/nginx.conf 文件的变化
- 有修改时对比变更内容，分析可能的影响
- 通知我修改了什么，是否有安全隐患
```

## 配置技巧

### 输出渠道选择

- **IM 推送**（钉钉/飞书/企微）：紧急通知、需要手机接收的
- **应用内对话**：不紧急但需要查看的
- **静默记录**：只需要留日志的

### Pre-check 机制

每次关切触发时，AI 会先判断「现在该不该执行」。比如日志监控关切，如果日志没有新的错误内容，AI 会自动跳过，避免发送无用通知。

### 有状态工作流

关切可以记住自己之前的状态。比如服务器巡检关切可以对比「上次检查时 CPU 是 50%，现在是 85%，明显升高」。
